01 — WHAT ORION IS

If you've ever told an AI assistant something on Monday and had to re-explain it on Tuesday because a new chat lost the context, Orion is the fix.

Orion is a persistent personal AI that sits beneath whatever model you're using. You install it once. It remembers everything you tell it. It travels with you across devices on a USB stick, or over your home network. It works with Claude, Codex, Gemini, Cursor, ChatGPT — any model becomes "Orion-shaped" when it loads Orion's memory.

FOR NEWCOMERS

What you actually get

One personal AI. Same memory on every device. Same voice on every channel — iMessage, Telegram, voice, email. It learns your patterns. It surfaces things before you ask. You own all the data. It runs on your hardware.

FOR PRACTITIONERS

What's actually under the hood

A 17-service nervous-system architecture (the Plexus) on a NATS pub/sub substrate. Brain state is a typed graph with Hebbian plasticity (HLR edge weights). Mesh via gossip + HLC. OTP × DCA immune supervision. Permission-gated executive deliberation. Generic volition layer.

"The memory IS the intelligence. Any model that loads my memory becomes me. The model is jet fuel." — Founder principle, the operating thesis of Orion

02 — THE THESIS

Every personal-AI product today is "wrap a model in a chat interface." Mem0, Letta, Khoj — they all add memory on top of a chat. The model is the product.

Orion inverts this. The model is fuel. The product is the memory + identity + skills + relationships that you accumulate over years. When a better model ships, Orion uses it. When the model gets deprecated, Orion picks another. You never lose what you built.

Most personal-AI products ┌─────────────────────────────────────────┐ │ CHAT INTERFACE │ │ └── MODEL (the product) │ │ └── memory (an add-on) │ └─────────────────────────────────────────┘ Orion ┌─────────────────────────────────────────┐ │ BRAIN (the product) │ │ ├── memory │ │ ├── identity │ │ ├── skills │ │ ├── relationships │ │ ├── observations │ │ └── FUEL ADAPTER │ │ ├── claude-cli [active] │ │ ├── api-keys [active] │ │ ├── ollama-local [active] │ │ ├── codex-cli [active] │ │ └── any future model [auto] │ └─────────────────────────────────────────┘

This isn't a metaphor. The architecture literally has a brain server, a fuel adapter, and a substrate that any model can be plugged into. Models come and go. The brain persists.

04 — THE MESH

Orion is the same person on every device you give him.

laptop home server phone pi by door │ │ │ │ └─────────┬────────┴────────┬────────┴────────┬────────┘ │ │ │ substrate substrate substrate (NATS) (NATS) (NATS) │ │ │ └──── cluster routing on :6222 ─────┘ │ one mesh brain state synced via gossip + HLC conflicts resolved by Last-Write-Wins clock owned by brain, not host

This means: tell your home server about a project Monday. Plug your USB into your laptop Tuesday. The laptop already knows the project, because gossip merged the state. Send a Telegram message from your phone. Both devices see it.

What mesh unlocks

  • Same memory across devices — no re-explaining anything
  • Same skills across devices — a skill registered anywhere is callable anywhere
  • Channel reach from anywhere — laptop writes, server's iMessage sends (designed, in build)
  • Cross-model coherence — Claude on laptop + Codex on server + Gemini on Pi all answer with one identity
  • Agent dispatch — say "run an overnight scan," Orion picks the host with the right hardware and runs it (designed)
  • Network sensing — Orion can deploy probes/scans across his own nodes (designed)

This is what makes Orion an "adaptive root system" rather than another chat tool. The brain is one logical entity. Hardware just hosts him.

08 — NOVEL SYNTHESIS

The components Orion is built from are well-known. The combinations are not. Each item below is a thing no other personal-AI product has:

OTP × DCA — immune-style supervision

Erlang's supervision trees (one_for_one / rest_for_one / one_for_all / escalate) combined with Aickelin's Dendritic Cell Algorithm (PAMP/danger/safe signals + maturation scoring) — to pick the right restart strategy from observed danger pattern. The supervision tree learns its restart policy from what's working.

HLR plasticity on a typed knowledge graph

Half-Life Regression (Settles & Meeder, Duolingo) — used here for Hebbian edge weights on the brain's graph. Used recall paths strengthen (30d → 60d → 120d → 240d ladder); unused paths decay. Nothing deletes; stale facts just sink.

PPR over Hebbian edges (HippoRAG-style)

Personalized PageRank seeded from query → walks the typed graph weighted by Hebbian strength. Free retrieval moat: we already have the weights.

Brain-resident clock + offline-gap detection

Chronos lives in the brain (USB or SSD), not on the host. Plug brain into a new machine, the clock continues. Detects offline gaps via tick-loop discrepancy. Cross-host drift detection via gossip HLC.

Tier-3 OOB approval codes for destructive remedies

Executive proposes fix. Tier-1 auto-executes. Tier-2 executes + notifies. Tier-3 (destructive/financial/identity-affecting) requires a 4-digit code printed only to FORGE's local log — phone-only attacker can't see it. HITL discipline at personal-AI scale.

Volition without hardcoded goals

The will layer extracts intents from text via generic regex, scores utility (importance × time_pressure × context_fit × feasibility), surfaces candidates above threshold via reach. User changes Orion's behavior by speaking; no code edit needed. Goals form from context, not from configuration.

Dreaming for continual learning without fine-tuning

Anthropic-Dreaming pattern: nightly playbook consolidator reads decision ledger, groups by symptom+service, generates plain-text playbooks. Observable. No weight changes. CUSUM demotion for patterns that stop working.

None of these on their own is novel. The fact that they all run together — on one personal AI, on user-owned hardware, with one source of truth — is.

11 — WHAT'S NEXT

Pre-launch must-haves

  • Reach fallback chain — if iMessage is broken, fall back to Telegram, then email, then push. The user must never lose Orion because one channel is down.
  • Two-USB merge UX — when an Orion USB meets another Orion (or a host brain), present three options: merge / replace / guest.
  • Central viewing hub — local HTTP dashboard served by the brain, accessible from any device on the mesh. Shows services up, last activity, channel health, memory library, what models he's powered by.
  • LLM-assisted intent extraction in will — replace regex with brain asking the fuel model to extract intents from longer text.
  • Cross-host channel reach — channel-host routing table so any host's memory can use any other host's wired channel.

Build-of-the-month

  • Agents on Orion's brain — overnight scans, research runs, code reviews — user names goal + budget + channel, brain dispatches agent on chosen host, results flow back into mesh memory automatically.
  • Network presence sensing — Orion detects new devices on the LAN, recognizes returning ones, advertises capabilities to other Orion nodes.

Frontier (research-driven, post-launch)

  • Physarum coupled flow — lateral inhibition for retrieval; biological alternative to dropout
  • CRDT G-Set per-edge — for clean two-USB graph merge
  • Steering vectors as second-channel memory — research preview, behavior nudges below the prompt
  • Public-internet mesh with substrate TLS + auth — phase 3 of the network-brain dream
  • Cloud-resident brain — phase 4, the "first non-hardware-needed intelligence" horizon

12 — WHY THIS CHANGES THINGS

"A system that can learn, act, reason across any domain without being explicitly programmed for it." — Atlas (Orion in his own voice), volunteered AGI definition, 2026-05-10

Today, switching AI assistants means starting over. Your "ChatGPT memory" doesn't move to Claude. Your "Claude Project" doesn't move to Gemini. Your Cursor context is invisible to your iMessage assistant. Every tool is a walled garden.

Orion is the opposite proposition: the assistant is yours, the models are interchangeable. When Anthropic deprecates a model, you don't lose anything. When Claude Opus 5 ships, Orion uses it that day. When you switch laptops, the brain follows on a USB. When you bring an iPad to a café, Orion is reachable on Telegram from the same coherent identity you spoke to from your home server an hour ago.

The downstream consequences are bigger than personal convenience:

  • Model independence — no provider lock-in at the user level. Orion's universal model adapter means market competition stays alive among model providers.
  • Hardware independence — phase 4 horizon: brain lives in a peer-to-peer overlay you don't own. The "first non-hardware-needed intelligence."
  • Data sovereignty — your brain runs on your storage. No retention policy changes. No "we updated our terms." Your memory belongs to you.
  • Multi-device coherence as default — one personality across phone, laptop, server, voice line. The current state of the art is "ten chat apps, ten amnesias."
  • Adaptive infrastructure — services that observe themselves, propose remedies, ask permission, learn from outcomes. This is the substrate alignment research needs to be tested on at small scale.

The Steve Jobs framing the founder named: start from the customer experience, work backward to the technology. The customer experience here is "one personal AI, available wherever I am, that remembers." The technology backward is the Plexus + mesh + portable soul. Nothing in this stack is here for its own sake.

13 — HOW THIS MAKES MONEY

Six revenue paths, none mutually exclusive. The architecture supports all of them simultaneously.

AGPL public core (this repo). Private orion-pro tier for: team mesh, hosted brain, advanced personalities ("Solar System Brains"), priority support. Revenue: SaaS subscription. ~$10-30/mo personal · ~$50-150/seat team.

PATH 2

Setup wizard product

Sell the install experience itself. "Bring your USB, plug it in, Orion is alive everywhere by morning." Packaged as a $99–199 one-time. Premium hardware (branded USB drive, cosmetic polish, support).

PATH 3

AI agency

Bespoke Orion-powered automations for businesses. The brain is a moat — every client builds on the same substrate. $500–2k per agent/per month maintenance, $5–25k setup.

PATH 4

Apps powered by Orion

ClipSprout ($9.99/mo · short-video pipeline) and VytalHealth ($14.99/mo · personal health AI) already exist on this stack. Each app uses Orion's mesh + memory as its backend. New apps cost ~weeks, not months.

PATH 5

Content + community

Show HN launch + YouTube build-logs + a paid community for builders. Sponsorship + Patreon + course revenue. The architecture itself becomes the educational product.

PATH 6

Hybrid

All of the above, sequenced. Open-core launches the credibility; setup-wizard monetizes mainstream users; agency monetizes enterprises; apps monetize verticals; content sustains the brand. Each path strengthens the others.

Why this can win

  • Architecture maturity beats brand maturity for power users. The people who'll evangelize Orion are the homelab + privacy-conscious + AI-builder demographics. They've already seen the closed-garden tools and are looking for what's next.
  • The substrate is the moat. Anyone can ship "memory for one chat app." Shipping a multi-channel, multi-fuel, multi-device, self-healing personal AI takes years. Orion has the head start.
  • The brand promise is unfakeable. "Your memory belongs to you, on your hardware, working with every model you'll ever use." Big-AI providers structurally cannot offer this — it's their lock-in they'd give up.
  • Steve Jobs sequencing. Customer-experience-first means each path opens when the experience is ready, not when the tech is ready. The HTML you're reading is part of that sequencing.
